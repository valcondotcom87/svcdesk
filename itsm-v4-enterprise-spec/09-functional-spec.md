# 09. Functional Specification

## Core Practices
- Incident Management
- Service Request Management
- Problem Management
- Change Enablement
- Service Level Management
- Knowledge Management
- Configuration Management (CMDB)
- Asset Management
- Service Catalog Management

## Role Model
- End User
- Service Desk Agent
- Incident Manager
- Problem Manager
- Change Manager
- CAB Member
- Asset Manager
- IT Director (read-only)
- Auditor (read-only + audit logs)

## Key Features by Practice

### Incident Management
- Impact/urgency matrix and auto-priority
- SLA timers and escalations
- Major incident workflow
- CI linkage and post-incident review

### Service Request
- Dynamic service catalog
- Multi-level approvals
- Automated fulfillment

### Problem Management
- Root cause analysis
- Known Error Database
- Linked incidents + workaround

### Change Enablement
- Standard/Normal/Emergency
- CAB workflow
- Risk scoring + change calendar
- CMDB impact mapping

### SLA & Reporting
- SLA policies per service
- OLA tracking
- KPI dashboards (MTTR, MTBF, FCR, SLA compliance)

### CMDB & Asset
- Relationship mapping
- Dependency visualization
- Lifecycle tracking and discovery API

### Knowledge
- Versioned articles
- Approval workflow
- Relevance scoring
